# AlphaBrief 路线图（Roadmap）

本路线图用于回应上一版实现（`run_agents.py`）可用但“工程化不足”的反馈，聚焦从“可跑”升级到“可维护、可验证、可扩展”。

## 目标与原则

- **可执行**：对外保持简单 CLI 体验。
- **可验证**：关键流程具备自动化测试。
- **可追溯**：每一步输出可审计、可复现。
- **可扩展**：后续可接入不同模型、不同合规策略。

---

## Phase 1（短期，1-2 周）：稳定性与可维护性

### 1.1 重构脚本结构
- 将 `run_agents.py` 拆分为模块（示例）：
  - `alphabrief/chunking.py`：分块策略
  - `alphabrief/prompting.py`：模板渲染
  - `alphabrief/workflow.py`：流程编排
  - `alphabrief/cli.py`：参数入口
- 保留兼容命令：`python3 run_agents.py --input ...`。

### 1.2 输入校验与错误处理
- 明确异常场景：文件不存在、编码错误、空文本、非法参数。
- 输出统一错误码（如 `2` 参数错误、`3` I/O 错误）。

### 1.3 测试补齐（优先）
- 新增单元测试覆盖：
  - 长度阈值是否触发分块
  - 分块边界（刚好阈值、超 1 字、空输入）
  - 模板占位符替换正确性
- 新增 CLI 集成测试：给定样本输入，断言关键段落输出存在。

**验收标准**
- 核心模块单测覆盖率达到基础阈值（建议 ≥80%）。
- CLI 在异常输入时返回非 0 并给出可读错误信息。

---

## Phase 2（中期，2-4 周）：质量与合规自动化

### 2.1 Chunking Agent 真正落地
- 当前版本只做“切块+提示”，下一步应支持：
  1) 对每个 chunk 生成中间摘要（可保存到文件）
  2) 合并中间摘要再进入最终总结阶段
- 增加 `--save-intermediate` 参数保存中间结果。

### 2.2 Compliance Guard 自动检查
- 将“合规清单”从打印提示升级为规则检测：
  - 检测买卖建议措辞
  - 检测价格目标表述
  - 检测保本/收益承诺表述
- 输出 `compliance_report.json`（规则、命中位置、风险等级）。

### 2.3 输出格式标准化
- 支持 `--output-format text|json`。
- JSON 模式包含字段：
  - `core_viewpoints`
  - `financial_data`
  - `logic_breakdown`
  - `risk_factors`
  - `executive_summary`
  - `disclaimer`

**验收标准**
- 在样例语料上能稳定产出结构化 JSON。
- 合规检测可定位高风险语句并给出明确标记。

---

## Phase 3（长期，4-8 周）：产品化与扩展

### 3.1 配置化模型与模板
- 新增配置文件（如 `config.yaml`）：
  - 模型名称、temperature
  - 最大字符阈值
  - 分块大小
  - 模板路径
- 支持多模板场景（行业研报、财报、会议纪要）。

### 3.2 可观测性与日志
- 为每次执行生成 run_id。
- 输出流程耗时、token 估算、阶段状态。
- 提供 `--verbose` 与 `--quiet` 模式。

### 3.3 交付形态优化
- 打包为可安装 CLI（如 `pip install -e .` 后使用 `alphabrief` 命令）。
- 增加 Docker 运行方式，降低环境差异。

**验收标准**
- 在干净环境中一条命令安装并运行。
- 日志可用于定位失败阶段与性能瓶颈。

---

## 建议里程碑

- **M1：基础稳态版**（完成 Phase 1）
- **M2：合规增强版**（完成 Phase 2）
- **M3：产品化版**（完成 Phase 3）

---

## 当前已知差距（相对上一版）

- 缺少自动化测试与模块化结构。
- Chunking 仅分割文本，尚未闭环“分块摘要→合并”。
- Compliance Guard 仅提示清单，尚未形成可执行检测。
- 输出缺少机器可读标准结构。

以上差距即本路线图的核心治理对象。
